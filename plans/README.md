## Findings considered and rejected

Recorded so the next audit doesn't re-litigate them:

- **"Listener leak" on `cw:cloud-data-changed` in recipe-browser.js:1558** — not a leak; this is a full-reload MPA and library.html reloads on bfcache restore (library.html:40), so listeners never accumulate.
- **"Stale lastPushed snapshot when cloud row missing" (sync.ts:755-768)** — by design; the guard's own comment explains it prevents the seed-upsert from being skipped. "Fixing" it would reintroduce a data-loss bug.
- **`realtimeSubscribedPromise` resolves only once (sync.ts:88-91)** — documented as a first-SUBSCRIBED test signal; intentional.
- **Dead script weight on minerals/taste pages** — false: minerals.html calls `deriveStockFormulaFromTarget` 6×; taste.html reaches diy-editor.js via mineral-selector.js. Only start.html's metrics.js tag was dead (handled in plan 005).
- **Paginate the public-library fetch (library-data.js:169)** — not worth doing at current catalog size (tens of rows, sessionStorage-cached). Revisit if public publishing grows past a few hundred rows.
- **`.env.test` exposure** — verified never committed and correctly gitignored; no action.
- **Creator-name fallback path in recipe-browser.js (476/652/1017)** — safe (`el()` uses textContent); cosmetic consistency only.
- **Implicit-grant OAuth tokens in deep-link hash (capacitor-bootstrap.ts:152)** — PKCE is the primary flow; native WebView mitigates referer leakage; no action.
- **Hoist repeated `getEffective*Sources()` calls in script.js `calculate()`** — storage load helpers are module-cached; negligible win, delicate function. Not worth the churn.
- **vite 5 → current major** — real but deliberately deferred; clears the remaining *moderate* audit findings (esbuild chain). Schedule as its own migration with the e2e suite as the gate (`*.spec.ts` runs against the built dist, which is exactly what a Vite major can break).
